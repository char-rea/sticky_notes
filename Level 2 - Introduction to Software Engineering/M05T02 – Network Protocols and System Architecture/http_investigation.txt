4. Connection workflow:
    Connection: keep-alive
- The client looks up the server’s IP address using the website name.
- A connection is opened to the server on port 443.
- A secure (HTTPS) connection is set up using encryption.
- The client sends an HTTP GET request to the server.
- The server receives the request and processes it.
- The server sends back a response with a status code, headers, and data.
- The connection stays open for reuse or is closed after the response.
